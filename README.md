## 📖 简介

**zhisi-skill**（织思写作助手）是一款专为人文学科（哲学、中国语言文学、艺术学、历史学等）论文写作设计的 AI 辅助工具。

它的核心理念是：人文社科学术写作的本质不是线性输出，而是将零散的文献阅读、跳跃的思考片段、反复的自我诘问 **"编织"** 成一个有机的论证整体。织思写作助手陪伴你完成这个编织过程。

### 它与通用 AI 写作工具的区别

| 维度 | 通用 AI 写作工具 | 织思写作助手 |
|------|-----------------|-------------|
| 目标用户 | 所有人 | 人文社科硕博研究生 |
| 核心能力 | 生成内容、语法修正 | 论证结构分析、逻辑检视、思维导航 |
| 写作模式 | 线性输入→输出 | 支持逆序写作、非线性跳写、草稿重组 |
| 分析深度 | 语言层面 | 逻辑谬误检测、学术对话感知、术语一致性 |
| 角色定位 | "替你写" | "陪你写，帮你检" |

### 不做什么

- ❌ 不做文献管理（建议使用 Zotero）
- ❌ 不做语法润色（建议使用 PaperPal / Grammarly）
- ❌ 不做查重降重
- ❌ 不替用户写论文（只分析、建议、指路）
- ❌ 不自动生成全文

## 🚀 安装

### Claude Code

```bash
# 克隆到本地
git clone https://github.com/li2990555-pixel/zhisi-skill.git
cd zhisi-skill

# 复制到 Claude Code Skills 目录
cp -r SKILL.md references scripts ~/.claude/skills/zhisi-skill/
```

然后通过 Claude Code 的 Skill 工具调用，或在对话中输入 `/zhisi-skill`。

### 跨平台（ChatGPT / Gemini / DeepSeek / Kimi）

直接将 [SKILL.md](./SKILL.md) 中的核心 Prompt 逻辑复制到你的对话中，作为系统提示词使用。所有核心功能均基于对话 Prompt 实现，不依赖特定平台能力。

## 🎯 使用场景

### 场景 1：开题阶段 — 搭建论文结构

> 小王是哲学系研二学生，刚确定论文选题。他打开织思，选择"谋篇→大纲设计"，输入导师修改过的残缺提纲。Skill 识别出论文类型为"比较型"，建议补充"理论框架"和"研究综述"两个关键部分，并生成完整的思维导图。

### 场景 2：写作中段 — 跳写失控后的补救

> 小李是中文系博士生，论文写了三个月，先写第三章又跳去写第五章。他感觉"走偏了"，选择"审视→查漏补缺"。Skill 扫描所有已写内容，标记出章节间的逻辑断裂，提供重组方案。

### 场景 3：初稿完成 — 全面检视

> 小张是艺术学硕士，初稿完成。选择"合诊"一键全面检视。Skill 检测出两处逻辑跳跃和一例诉诸权威，并生成写作健康度仪表盘，标注最需要优先处理的问题。

## 🧠 设计理念

### 人文学科的写作本质

不同于理工科论文的"方法→数据→结论"范式，人文学科论文的核心是**论证链条**：概念是否清楚、理论是否能解释材料、论证是否严密。织思写作助手的设计围绕这一核心差异展开。

### 非线性写作支持

人文学者很少按照"第一章→第二章→第三章"的顺序写作。真实的写作过程是跳跃的、缝补的、反复的。织思尊重这一习惯，提供逆序写作引导和草稿重组功能，帮助用户在跳跃中不丢失整体控制。

### 辅助而非替代

织思的定位是"思维伴侣"——分析、建议、指路，但不替用户写论文。这种定位既是道德选择，也是产品差异化的核心。
