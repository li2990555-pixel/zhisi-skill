# zhisi-skill — 织思写作助手

**人文社科学术论文写作 Skill（非线性版）**

专为真实写作过程设计的论文辅助工具。不教你怎么写，而是适应你怎么写。

如果你写论文不是从头到尾线性推进的——先写中间章节，最后写前言，思路来回跳跃，找到好材料先存着但之后忘了——这个工具就是为你设计的。

---

## 这和其他论文工具有什么不同？

绝大多数论文工具假设你是线性写作的：先写提纲 → 再写前言 → 然后一章一章往下推。但很多人不是这样写的。

这个 skill 不试图"纠正"你的写作方式。它做五件独特的事：

### 五件事

| 功能 | 是什么 | 为什么特别 |
|------|--------|-----------|
| 🗺️ **谋篇** | 大纲设计 + 逆序写作引导 + 非线性草稿重组 | 你跳着写，它如实反映跳跃，但帮你重建逻辑。支持先写结论再反推导论 |
| 🔍 **审视** | 查漏补缺 + 论证强度分析 + 术语一致性 + 学术对话感知 | 不只是"空章节提醒"，而是真正的学术判断——检测 8 类逻辑谬误 |
| 📊 **览局** | 写作健康度仪表盘 | 看一眼就知道论文全貌：结构、连贯性、引用、论证、术语、进度 |
| 📦 **积材** | 材料库梳理 + 跨学科视角推荐 | ADHD 友好——只管往里丢，写的时候 AI 自动提醒你用上 |
| ✍️ **行文** | 批注式写作 + 写作障碍突破 | 写作过程中实时反馈，卡壳时给你换条路走 |

## 核心机制

### 🗺️ 谋篇 — 你的论文不是一条线，而是一片正在绘制的地形

**大纲设计**

输入你的提纲（残缺、无序都可以）。AI 识别论文类型——思辨型、历史型、比较型还是实证型——然后补全缺失部分，标注每章之间的逻辑承接关系，输出完整的思维导图。

**逆序写作引导**

先写了结论？没问题。把结语给 AI，它提取核心论点，反向推导导论需要铺设的问题链：

> "你的结论得出'庄子逍遥并非绝对自由'，那么导论需要：①梳理自由概念的哲学史 ②界定论域边界"

**非线性草稿重组**

你在不同章节之间跳着写，写完发现逻辑是散的。把碎片给 AI，它会分析每一段的论证功能和依赖关系，生成 1–3 种重组方案，标注每种方案的逻辑优势和代价。

---

### 🔍 审视 — 三类检视，不只是查错

**查漏补缺**

追踪已写内容与大纲的差距。不要求按顺序写，AI 自动跟踪每章每节的内容覆盖，标记：

- 🔴 缺失 ("第三章还未写")
- 🟡 浅覆盖 ("第一章只写了开头")
- 🟢 已覆盖 ("第五章完成度良好")

**论证强度分析**

AI 逐句检测 8 类逻辑谬误，定位到具体句子：

| 谬误 | 示例 |
|------|------|
| 循环论证 | "A因为B，B因为A" |
| 偷换概念 | "自由"前半段指政治自由，后半段变成精神自由 |
| 稻草人 | 弱化对方论点再驳斥 |
| 不当类比 | 将 A 理论与 B 现象机械对应 |
| 以偏概全 | 单一文本分析 → 断代特征 |
| 诉诸权威 | "正如福柯所说"直接作为结论 |
| 逻辑跳跃 | 现象描述 → 直接跳到结论 |
| 虚假两难 | "要么是X要么是Y" |

输出定位到具体句子的分析报告：

> 🚩 第3章第2段：**逻辑跳跃**
> 原文："庄子的逍遥游体现了自由精神，因此我们可以用存在论来理解。"
> 从"自由精神"直接跳到"存在论"，建议补充过渡环节。

**术语一致性检查**

跨章节扫描核心术语，识别译名不一致（"德里达" vs "德希达"）、含义漂移、中英夹杂。

**学术对话感知**

AI 分析你文中涉及的学者、理论、流派，标记对话状态：

- 🔵 继承——"延续了伽达默尔的诠释学路径"
- 🟣 批判——"与福柯对话但未回应该领域的关键反驳"
- ⚪ 空白——"此概念已有重要讨论，建议纳入对话"
- ❌ 伪对话——"引用某学者但未与之真正学术对话"

### 📦 积材 — 材料库 + 跨学科视角

**材料库梳理**

把论文相关的材料（引用、笔记、灵感片段、待定素材）丢进去。AI 自动识别文献类型、关键词，判断可能适合放在哪个章节。

**跨学科视角推荐**

你写"庄子自由观"？AI 推荐政治哲学的柏林"两种自由概念"、心理学的自我决定理论、比较哲学的跨文化方法论，同时标注引用注意事项。

---

### ✍️ 行文 — 批注式写作 + 写作障碍突破

**批注式写作**

写作过程中粘贴段落，AI 逐句批注：

> 原文："庄子的逍遥游体现了某种自由精神，这对现代人有重要启示。"
>
> ⚡ 论证偏弱 → "某种自由精神"表述笼统
> 📚 缺乏引用 → "重要启示"需要文献支撑

**写作障碍突破**

卡壳时，AI 给你四条出路：换起点、换视角、换方法、对话法。选一个，继续写。

## 使用场景

**场景一：跳着写，结构散**

你在写第三章的时候用了一个概念，但理论框架那一章还没写。AI 不会阻止你，但会提醒：

> "你直接使用了'战术'这个概念，但理论框架部分还没有正式界定。建议先做一个简要的概念说明，或者回头写理论框架部分。"

**场景二：初稿写完了，心里没底**

选择合诊（F），AI 运行全面检视：论证强度、术语一致性、查漏补缺、学术对话感知、冗余检测……汇总一份报告，标注最需要优先处理的问题。

**场景三：先写了结论**

你把论文结语贴进去。AI 提取三个核心论点，反向生成导论需要铺设的问题链——每个结论对应什么前置问题，一目了然。

**场景四：存了一堆材料不知道放哪**

把零散的引用、笔记、片段存进材料库。AI 自动分类，等你写到相关内容时主动提醒：

> "你之前存过一段德塞托讲'战术'的原文，放在第三节开头作为概念界定很合适。"

## 安装

### Claude Code（原生支持）

```bash
# 克隆
git clone https://github.com/li2990555-pixel/zhisi-skill.git
cd zhisi-skill

# 安装到 skills 目录
cp -r SKILL.md references scripts ~/.claude/skills/zhisi-skill/
```

使用：在对话中输入 `/zhisi-skill` 或"帮我分析一下论文论证"。

### ChatGPT / DeepSeek / Gemini / Kimi 等平台

这些平台没有 Skill 系统，需使用精简版 `SKILL-lite.md`（已剥离 Claude 专属指令，纯 Prompt 格式）。

1. 下载 `SKILL-lite.md`
2. 将全部内容粘贴到平台的自定义指令 / 系统提示词中
3. 开始对话即可使用

> ⚠️ 与 Claude Code 版的区别：核心分析能力一致（论证检测、大纲设计、仪表盘等），但无法自动读写文件和并行分析。请手动分章节输入，建议将分析结果复制到笔记中保存。

## AI 行为规范

启用本 skill 后，AI 按三层结构工作：

**第一层：实时检测（不打断）**

回答你的同时隐性运行：内容检视、逻辑扫描、语料匹配。

**第二层：适时提醒**

- 检测到严重问题（概念未界定、逻辑谬误）→ 立即提醒
- 常规问题 → 相关话题出现时温和提示
- 材料库匹配度高 → 主动问你要不要用

**第三层：用户触发**

`C` 审视 / `F` 合诊 → 全面扫描，输出报告。

所有提醒是建议性的。最终决定权在你。

## 设计原则

- **适应你，而非改造你** — 你按自己的方式写，AI 来适应你的节奏
- **协助而非纠正** — 所有提醒都是建议性的，最终决定权在你
- **专注人文社科** — 检测规则和判断依据专门针对人文社科学术写作设计
- **不替写** — 分析、建议、指路，但不生成正文
