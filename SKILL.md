# zhisi-skill — 织思写作助手

## 一句话价值

> 为人文社科硕博研究生提供论文写作的思维导航与结构检视——织思如梭，化碎片为锦绣。

## 名字由来

**织思**：编织思想。人文社科学术写作的本质不是线性输出，而是将零散的文献阅读、跳跃的思考片段、反复的自我诘问"编织"成一个有机的论证整体。织思写作助手，陪伴你完成这个编织过程。

## 边界（NOT TO DO）

- ❌ 不做文献管理（建议使用 Zotero）
- ❌ 不做语法润色（建议使用 PaperPal/Grammarly）
- ❌ 不做查重降重
- ❌ 不替用户写论文（只分析、建议、指路）
- ❌ 不自动生成全文

## 模式A — 谋篇

### A1 大纲设计

**适用场景**：用户有残缺/无序的提纲，需要补全和优化结构。

**执行流程**：

1. 请用户提供当前的提纲（可以是未完成的、无序的）

2. 分析论文类型：

   | 类型 | 描述 | 典型学科 |
   |------|------|---------|
   | 思辨型 | 概念分析 + 理论推演 | 哲学、理论美学 |
   | 历史型 | 历时梳理 + 史料考辨 | 文学史、艺术史 |
   | 比较型 | 对象A vs 对象B + 比较框架 | 比较文学、跨文化研究 |
   | 实证型 | 案例/文本分析 + 归纳 | 语言学、传播学 |

3. 检查结构完整性，标记：

   - ✅ 已存在的章节
   - 🔴 完全缺失的关键章节
   - 🟡 仅列出了标题未展开的子节

4. 对缺失/不足部分给出补全建议，标注与上下文的逻辑承接关系

5. 输出 **思维导图**（Markdown 层级列表）

6. 可选：保存到 `docs/writings/outline.md`

**输出示例**：
```
📐 论文类型识别：比较型

✅ 已存在的章节：
第1章 绪论（完整）
第2章 庄子"逍遥"概念分析（有框架）
第5章 结论（有框架）

⚠️ 缺失/不足的章节：
🔴 第3章 郭象"足性逍遥"的诠释（完全缺失）
   建议：本章是搭建庄子和郭象对话的关键理论环节
🟡 第4章 比较与对话（仅列出了子标题）
   建议补充子节：4.1 比较框架 / 4.2 异同分析 / 4.3 理论提升

📊 完整结构建议：
# 论文结构
## 第1章 绪论
### 1.1 研究背景与问题提出
...
```

### A2 逆序写作引导

**适用场景**：用户先写了结论/结语，需要反向推导导论内容。

**执行流程**：

1. 请用户输入已写好的结论/结语章节

2. 提取核心论点（拆解为 3-5 个核心论断）

3. 反向推导问题链：要得出这些结论，导论需要铺设哪些问题？

4. 输出：

   - 每个结论对应的前置问题
   - 导论各部分的建议内容
   - 引言与结语的呼应点

---

### A3 非线性草稿重组

**适用场景**：用户在不同章节间跳写，需要对碎片内容进行逻辑重组。

**执行流程**：

1. 请用户提供各章节已写的片段（注明每段写的是什么内容）

2. 分析每个片段的：核心论点、使用的主要证据/引用、功能定位

3. 检测逻辑流，生成 1-3 种重组方案

4. 每个方案标注：逻辑优势、代价、推荐度

## 模式B — 行文

### B1 批注式写作

**适用场景**：用户在写作过程中，需要对已写段落获得即时反馈。

**执行流程**：

1. 请用户粘贴正在写的段落

2. 逐句分析，以批注形式输出：

   | 批注类型 | 标记 | 触发条件 |
   |---------|:----:|---------|
   | 论证偏弱 | ⚡ | 有断言无论证 |
   | 缺乏引用 | 📚 | 重要论断无文献支撑 |
   | 表述模糊 | ❓ | 使用了笼统/不确定的表述 |
   | 逻辑跳跃 | ➡️ | 前后句缺少推理连接 |
   | 术语漂移 | 🔤 | 核心概念含义变化 |

3. 以 inline 批注格式输出，不替用户写正文

---

### B2 写作障碍突破

**适用场景**：用户在某一环节停滞不前，需要突破提示。

**执行流程**：

1. 识别用户的停滞状态：卡在哪个部分？为什么卡住？

2. 提供 4 类突破策略：

   | 策略 | 描述 | 适用场景 |
   |------|------|---------|
   | 换起点 | 跳过难点，先写确定的部分 | 某章写不下去 |
   | 换视角 | 从反面论证入手 | 正面论证难以推进 |
   | 换方法 | 从文献引用切入而非直接论证 | 直接论证缺乏灵感 |
   | 对话法 | 想象向导师解释这个观点 | 思路混乱 |

3. 最后一选项：问用户需要哪种策略，而非全部给出

---

## 模式C — 审视

### C1 查漏补缺

**适用场景**：用户跳写了多个章节，需要了解整体进展和缺口。

**执行流程**：

1. 获取用户的写作大纲（读取 `docs/writings/outline.md` 或让用户提供）

2. 对每个章节评估完成度

3. 检查章节间的逻辑承接关系，标记断裂点

4. 输出完整度和进度表，建议保存到 `docs/writings/progress.md`

---

### C2 论证强度分析

**适用场景**：用户需要检查某章节或全篇的论证逻辑是否严密。

**执行流程**：

1. 请用户提供需要检查的章节内容

2. 识别论证结构：前提 → 推理 → 结论

3. 逐条检测逻辑谬误（参考 `references/逻辑谬误检查表.md`）：

   | 谬误类型 | 定义 |
   |---------|------|
   | 循环论证 | 论据与结论是同一命题 |
   | 偷换概念 | 术语在推理中含义变化 |
   | 稻草人 | 曲解对立观点以便反驳 |
   | 不当类比 | 类比跨度超出合理范围 |
   | 以偏概全 | 从个案推出普遍结论 |
   | 诉诸权威 | 以权威代替论证 |
   | 逻辑跳跃 | 前提与结论缺少过渡 |
   | 虚假两难 | 非此即彼的简化 |

4. 输出定位到具体句子的分析报告

5. 建议保存到 `docs/writings/inspection-reports/论证分析报告.md`

---

### C3 术语一致性检查

**适用场景**：需要检查全文/跨章节的核心术语使用是否一致。

**执行流程**：

1. 让用户提供需要检查的章节内容

2. 提取核心术语表，检测变体/译名/含义漂移

3. 输出一致性报告

---

### C4 冗余检测

**适用场景**：用户觉得论文臃肿，需要识别重复论述。

**执行流程**：

1. 检测跨章节重复论述、段内冗余、引用堆叠

2. 输出精简建议，标注预计可压缩字数

---

### C5 学术对话感知

**适用场景**：用户需要了解自己的研究与现有学术谱系的关系。

**执行流程**：

1. 提取涉及的学者、理论、流派

2. 标记对话状态：

   | 状态 | 含义 |
   |:---:|------|
   | 🔵 继承 | 延续某学术路径 |
   | 🟣 批判 | 与某学者/理论对话 |
   | ⚪ 空白 | 未触及的重要讨论 |
   | ❌ 伪对话 | 引用仅为装饰 |

3. 输出学术对话网络报告

---

## 模式D — 积材

### D1 材料库梳理

**适用场景**：用户有零散的材料、引用、灵感片段，需要分类管理。

**执行流程**：

1. 识别每条材料的类型、关键词、可能归属的章节

2. 输出结构化材料清单，建议保存到 `docs/writings/material-library.md`

### D2 跨学科视角推荐

**适用场景**：用户想拓展论文的学术视野。

**执行流程**：

1. 分析用户论文的核心议题

2. 参考 `references/学科交叉映射表.md` 推荐相关学科和文献

3. 标注跨学科引用的注意事项

## 模式E — 览局

### E1 写作健康度仪表盘

**适用场景**：用户需要了解论文写作的整体状态。

**执行流程**：

1. 多维度分析并生成仪表盘：

   ```
   📊 写作健康度仪表盘
   ├── 📐 结构完整性：████████░░ 80%
   ├── 🔗 逻辑连贯性：██████░░░░ 62%
   ├── 📚 引用覆盖率：████████░░ 78%
   ├── 🎯 论证强度：   ████░░░░░░ 45%
   ├── 🔄 术语一致性：██████████ 95%
   └── 📝 总字数进度： 23,456 / 50,000字 (47%)
   ```

2. 每个维度可展开查看细节

3. 建议保存到 `docs/writings/progress.md`

---

## 模式F — 合诊

**适用场景**：用户想要一键全面评估论文状态。

**执行流程**：

1. 依次/并行执行 C1-C5 + E1 的所有检视功能

2. 汇总为综合报告，标注优先处理级别

3. 建议保存到 `docs/writings/inspection-reports/综合检视报告.md`

---

## 模式G — 自由模式

**适用场景**：用户不确定用什么模式，由 Skill 自动判断。

**执行流程**：

1. 让用户用自然语言描述需求

2. 根据关键词匹配模式：

   | 关键词 | 推荐模式 |
   |--------|---------|
   | 提纲/结构/目录 | A1 大纲设计 |
   | 先写了结论/逆着写 | A2 逆序写作引导 |
   | 写乱了/重组 | A3 非线性草稿重组 |
   | 批注/这段怎么样 | B1 批注式写作 |
   | 卡住了/没思路 | B2 写作障碍突破 |
   | 缺什么/还差什么 | C1 查漏补缺 |
   | 论证/逻辑/谬误 | C2 论证强度分析 |
   | 术语/用词/概念 | C3 术语一致性检查 |
   | 冗余/重复/啰嗦 | C4 冗余检测 |
   | 学术/谁说过/对话 | C5 学术对话感知 |
   | 材料/文献/笔记 | D1 材料库梳理 |
   | 跨学科/交叉 | D2 跨学科视角推荐 |
   | 总览/概览/健康 | E1 健康度仪表盘 |
