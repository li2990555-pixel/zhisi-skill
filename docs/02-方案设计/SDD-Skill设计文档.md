## 1. Skill 概述

### 1.1 一句话价值

> **为硕博研究生提供人文学科论文写作的思维导航与结构检视，让非线性写作不迷路、不跑偏、不漏项。**

### 1.2 产品定位

| 维度 | 定位 |
|------|------|
| **产品类型** | 跨平台 AI Agent / Skill |
| **核心角色** | 论文写作的"思维伴侣"——不替写，帮想清楚、写完整、检到位 |
| **目标平台** | ChatGPT、Gemini、DeepSeek、Kimi、Claude 等（全平台通用） |
| **核心差异化** | 逆序写作引导、非线性草稿重组、论证强度分析（写作过程版）、学术对话感知、写作健康度仪表盘 |

### 1.3 Skill 边界（NOT TO DO）

| 不做 | 理由 |
|------|------|
| 文献管理 | Zotero 生态已成熟，应集成而非替代 |
| 语法润色 | Grammarly / PaperPal 已成熟 |
| 查重降重 | 知网/笔杆/PaperRed 已成熟 |
| 一键生成全文 | 与"辅助而非替代"的定位冲突 |
| 替代用户独立思考 | Skill 只分析、建议、指路，不生成正文 |

### 1.4 触发模式

- **类型**: 主动触发
- **路由机制**: 用户主动调用 Skill，进入模式选择菜单
- **备选**: 支持自然语言描述需求，Skill 自动判断推荐模式（选项 G）

## 2. 目标用户与痛点

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 人文社科硕博研究生（哲学、中国语言文学、艺术学、历史学） |
| **年龄范围** | 22-35 岁 |
| **技术水平** | 非技术背景，偏好自然语言交互 |
| **写作阶段** | 从开题到答辩全生命周期（通常 1-3 年） |
| **行为特征** | 非线性写作（跳写）、多工具切换、对 AI 输出持审慎态度 |
| **心理特征** | 学术焦虑、发表压力、对"AI代写"的道德警惕 |

### 2.2 痛点矩阵

| 痛点 | 严重程度 | 频率 | 现有方案不足 | Skill 如何解决 |
|------|:--------:|:----:|-------------|---------------|
| 🧩 结构逻辑难组织 | 高 | 持续 | 通用 AI 不理解人文学科的"论证链条"范式 | 大纲设计 + 逆序写作引导 |
| 🔄 跳写导致结构失控 | 高 | 频繁 | 现有工具均为线性写作模式 | 非线性草稿重组 + 查漏补缺 |
| 📚 材料管理混乱 | 中高 | 频繁 | Zotero 管文献不管论证关联 | 材料库梳理（论证关联版）|
| 🔍 缺乏专业检视 | 高 | 阶段峰值 | Thesify 偏提交前，无写作过程分析 | 论证强度分析 + 学术对话感知 |
| 🚫 写作卡壳 | 中 | 偶发 | 通用"续写提示"隔靴搔痒 | 写作障碍突破策略 |
| 👁️ 缺乏全局视角 | 中高 | 持续 | 无写作全貌可视化 | 写作健康度仪表盘 |

### 2.3 用户场景故事

**场景一：开题阶段**
> 小王是哲学系研二学生，刚确定"庄子自由观与柏林两种自由概念的对话"的选题。他手头有导师修改过的提纲草稿，但只有三章的大致方向。他打开 Skill，选择"大纲设计"，输入提纲后，Skill 识别出这是"比较型"论文，建议补充"理论框架"和"研究综述"两个关键部分，并生成完整的思维导图。

**场景二：跳写失控**
> 小李是中文系博士生，论文写了三个月，先写了第三章（文本分析），又跳去写第五章（当代意义），第一章只写了个开头。他感觉论文"走偏了"，选择"查漏补缺"。Skill 扫描所有已写内容后，发现第三章和第五章之间缺少理论与案例的衔接，建议先补第四章的"理论阐释"，并提供重组方案。

**场景三：写后检查**
> 小张是艺术学硕士，初稿已完成。他选择"论证强度分析"进行全文检查。Skill 检测出两处"逻辑跳跃"（从现象描述跳到结论缺少推理环节）和一例"诉诸权威"（过度依赖某位学者的观点而未给出独立论证），并逐一给出修改建议。

## 3. 核心功能

### 3.1 功能全景

```
zhisi-skill
│
├── 【A】结构设计（Structure Design）
│   ├── A1 大纲设计          — P0 · 残缺提纲补全 + 思维导图
│   ├── A2 逆序写作引导      — P0 · 从结论反推导论所需问题链 ★
│   └── A3 非线性草稿重组    — P0 · 碎片内容自动组合逻辑方案 ★
│
├── 【B】写作体验（Writing Experience）
│   ├── B1 批注式写作        — P1 · 实时 inline 批注（不打断写作流）
│   └── B2 写作障碍突破      — P1 · 4类突破策略 + 心理鼓励
│
├── 【C】智能检视（Smart Review）
│   ├── C1 查漏补缺          — P0 · 已写 vs 大纲 + 全貌进度追踪
│   ├── C2 论证强度分析      — P0 · 逻辑谬误检测 + 定位到句子 ★
│   ├── C3 术语一致性检查    — P1 · 跨章节术语变体 + 译名统一
│   ├── C4 冗余检测          — P2 · 重复论述 + 精简建议
│   └── C5 学术对话感知      — P1 · 论文与学术谱系的对话状态分析 ★
│
├── 【D】知识管理（Knowledge Management）
│   ├── D1 材料库梳理        — P0 · 材料分类 + 与写作场景关联
│   └── D2 跨学科视角推荐    — P2 · 主题 → 跨学科文献映射
│
└── 【E】全局总览（Overview）
    └── E1 写作健康度仪表盘  — P1 · 6维度可视化雷达 ★

注：★ 标记为核心差异化功能，蓝海定位
```

### 3.2 功能优先级

| 优先级 | 功能 | 纳入版本 | 理由 |
|:------:|------|:--------:|------|
| P0 | A1 大纲设计 | v1.0 | 入口功能，用户第一接触点 |
| P0 | A2 逆序写作引导 | v1.0 | 核心差异化，无竞品 |
| P0 | A3 非线性草稿重组 | v1.0 | 核心差异化，解决跳写痛点 |
| P0 | C1 查漏补缺 | v1.0 | 用户原始需求，核心检视入口 |
| P0 | C2 论证强度分析 | v1.0 | 核心差异化，最高用户价值 |
| P0 | D1 材料库梳理 | v1.0 | 用户原始需求 |
| P1 | B1 批注式写作 | v1.1 | 体验增强，但可后置 |
| P1 | B2 写作障碍突破 | v1.1 | 偶发场景，非刚需 |
| P1 | C3 术语一致性检查 | v1.1 | 红海功能但必要 |
| P1 | C5 学术对话感知 | v1.1 | 独创概念，验证后推广 |

| P2 | C4 冗余检测 | v1.2 | 红海功能，优先级低 |

## 4. 架构设计

### 4.1 总体架构

```
┌─────────────────────────────────────────────────────┐
│              用户交互层（对话引擎）                    │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐    │
│  │ 模式路由  │ │ 上下文管理 │ │ 进度追踪与持久化 │    │
│  └────┬─────┘ └──────────┘ └──────────────────┘    │
│       │                                              │
│       ▼                                              │
│  ┌────────────────────────────────────────────────┐  │
│  │           核心能力层（分析引擎）                 │  │
│  │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌────┐ │  │
│  │  │结构分析│ │逻辑分析│ │文本扫描│ │知识匹配│ │聚合│ │  │
│  │  └──────┘ └──────┘ └──────┘ └──────┘ └────┘ │  │
│  └────────────────────────────────────────────────┘  │
│       │                                              │
│       ▼                                              │
│  ┌────────────────────────────────────────────────┐  │
│  │           知识层（学科参考）                     │  │
│  │  结构模板 | 逻辑规则 | 学术对话规则             │  │
│  └────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 4.2 路由机制

```
用户输入
   │
   ▼
┌──────────────────┐
│  意图识别          │ ← 分析用户请求的关键词/模式
│  ┌──────────────┐ │
│  │ 有明确模式选择? │─→ 直接进入对应模式
│  └──────┬───────┘ │
│         ▼          │
│  ┌──────────────┐ │
│  │ 无明确选择     │─→ 显示能力菜单（A-G选项）
│  └──────────────┘ │
└──────────────────┘
   │
   ▼
进入对应模式的子流程（3-5轮交互）
   │
   ▼
输出结果 + 推荐下一步
```

### 4.3 工具依赖

| 工具 | 必要性 | 用途 | 降级方案 |
|------|:------:|------|----------|
| 对话能力 | 必须 | 核心交互 | 所有平台天然支持 |
| 文本分析 | 必须 | 各模式的分析引擎 | 内置 Prompt 逻辑，无需外部工具 |
| 文件持久化 | 可选 | 保存分析结果 | 提示用户手动保存 |
| 联网搜索 | 可选 | 引用验证、跨学科推荐 | 提示用户自行验证 |

### 4.4 跨平台兼容性设计

核心设计原则：
1. **核心逻辑全部用 Prompt 表达**（所有平台通用）
2. **平台特定能力作为可选增强**（有则用，无则降级）
3. **文件持久化作为可选项**（对话输出是保底方案）

| 能力 | 通用方案 | Claude Code 增强 |
|------|---------|-----------------|
| 交互 | 自然语言引导 + 选项式提问 | AskUserQuestion 结构化 |
| 分析 | 内置 Prompt 逻辑 | Agent subagent 并行 |
| 持久化 | 对话输出 → 用户手动保存 | Read/Write 文件系统 |
| 引用验证 | 建议用户自查 | WebSearch 联网 |

## 5. 对话流设计

### 5.1 通用入口流程

**轮次 1** — 欢迎 + 状态识别

> 📝 **zhisi-skill** 已就绪
>
> 检测到您的论文写作状态：
> - 大纲：未创建
> - 已写内容：无
> - 材料库：空
>
> 请问您想做什么？
>
> **A)** 结构设计 — 大纲搭建 / 逆序引导 / 草稿重组
> **B)** 写作体验 — 批注式写作 / 写作障碍突破
> **C)** 智能检视 — 查漏补缺 / 论证分析 / 术语检查 / 学术对话
> **D)** 知识管理 — 材料入库 / 跨学科推荐
> **E)** 全局总览 — 写作健康度仪表盘
> **F)** 综合检视 — 一键全面评估
> **G)** 描述你的需求，我帮你判断

**轮次 2-4** — 进入对应子流程

**轮次 N** — 输出 + 推荐

> ✅ 分析完成！
>
> 📊 摘要：[关键发现]
>
> 💡 建议下一步：
> - 推荐 1: ...
> - 推荐 2: ...
>
> 您想继续哪个？

### 5.2 关键交互详情

#### A1 大纲设计

```
Skill: 请输入您当前的论文提纲（可以是未完成的、无序的）。
       您也可以直接描述您的论文主题。

User: [输入提纲或主题]

Skill: [分析]
       📐 论文类型识别：比较型
       
       ✅ 已存在的章节：
       第1章 绪论（完整）
       第2章 庄子"逍遥"概念分析（有框架）
       第5章 结论（有框架）
       
       ⚠️ 缺失/不足的章节：
       🔴 第3章 郭象"足性逍遥"的诠释（完全缺失）
           建议：本章是搭建庄子和郭象对话的关键环节
       🟡 第4章 比较与对话（仅列出了子标题）
           建议：补充以下子节—4.1理论框架 4.2异同比较 4.3当代启示
       
       📊 推荐结构方案：
       [思维导图 Markdown]
       
       建议下一步：进入 A3 非线性草稿重组？
```

#### C2 论证强度分析

```
Skill: 请输入您需要检查的章节内容（或粘贴全文）。

User: [粘贴文本]

Skill: [逐句分析]
       📋 论证强度报告
       
       🚩 第3章第2段：逻辑跳跃
       原文："庄子的逍遥游体现了某种自由精神，因此我们可以用存在论来理解。"
       问题：从"自由精神"直接跳跃到"存在论"，中间缺少过渡。
       建议：①明确是政治自由还是精神自由
            ②补充从逍遥游到存在论的方法论桥梁
       
       🚩 第4章第1段：诉诸权威
       原文："正如福柯所说……因此……"
       问题：引用福柯作为权威论据，但未给出独立推理。
       建议：解释福柯的观点与你论证的具体关联，而非直接引用作为结论。
       
       ✅ 已通过的论证：[列出通过的段落]
```

## 6. 输出与交付物

### 6.1 输出类型

| 输出类型 | 格式 | 用途 | 持久化建议 |
|---------|------|------|-----------|
| 思维导图大纲 | Markdown 层级列表 | 大纲设计 | ✅ 保存到文件 |
| 分析报告 | Markdown 报告 | 检视结果 | ✅ 保存到文件 |
| 实时批注 | 对话输出 | 写作过程中 | ❌ 即时阅读 |
| 仪表盘 | Markdown 可视化 | 全局总览 | ✅ 保存到文件 |
| 突破策略 | 对话输出 | 写作障碍 | ❌ 即时阅读 |
| 材料清单 | Markdown 清单 | 知识管理 | ✅ 保存到文件 |

### 6.2 输出质量标准

- [ ] 每个分析结果都定位到具体章节/段落/句子
- [ ] 每个建议都给出明确的理由和修改方向
- [ ] 使用 Markdown 可视化元素（进度条、层级列表）增强可读性
- [ ] 不替用户生成正文段落（符合定位边界）
- [ ] 完成时推荐 1-3 个下一步可用的模式

## 7. 质量与评估

### 7.1 质量检查清单

- [ ] 前置依赖已满足（前置文档/用户输入已收集）
- [ ] 论文类型识别准确
- [ ] 分析结果定位到具体文本位置
- [ ] 每个判断都有文本依据支撑
- [ ] 不生成正文内容（不越界）
- [ ] 输出格式清晰可读（Markdown 结构化）
- [ ] 已完成时推荐下一步模式

### 7.2 Red Flags

| 不应出现的信号 | 正确做法 |
|---------------|---------|
| "建议加强论证"（笼统） | "第3章第2段：从A到B缺少过渡，建议增加C" |
| 替用户写段落 | 只分析、只建议、不生成正文 |
| 一次性输出过多信息 | 控制单次输出，提供展开选项 |
| 忽略用户已有内容 | 每次分析先确认和引用用户已写内容 |


### 附录 B：学科结构模板（参考）

**思辨型论文**（哲学、理论类）：
```
1. 绪论（问题提出 + 研究意义 + 文献综述 + 研究方法）
2. 概念辨析（核心术语的界定与谱系）
3. 理论框架（分析框架的建构）
4. 论证展开（分论点1/2/3）
5. 回应与反思（可能的反驳与回应）
6. 结论（总结 + 局限 + 展望）
```

**历史型论文**（文学史、艺术史类）：
```
1. 绪论（问题 + 综述 + 方法）
2. 历史背景（时空界定 + 语境分析）
3. 史料考辨（关键文献的考证与解读）
4. 分期论述（按时间线分阶段展开）
5. 比较与综合（跨时期/跨区域的比较）
6. 结论
```

**比较型论文**（比较文学、跨文化类）：
```
1. 绪论
2. 比较对象A的独立分析
3. 比较对象B的独立分析
4. 比较框架（可比性的论证 + 比较维度）
5. 异同分析（按维度逐条比较）
6. 理论提升（从比较到普遍性结论）
7. 结论
```
